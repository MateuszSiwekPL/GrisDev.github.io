Fingers Testing

  • Genre: Skill Game
  • In Game Services: Ads (Banner, Interstitial)
  • Link: play.google.com/store/apps/details?id=com.DefaultCompany.TestYourFingers
Fingers Testing was my firs available to download project. It’s made of 5 minigames that will test players: reaction time, speed, accuracy, endurance and sliding speed. At first I was making it for learning purposes but later decided to release it on Google Play Store because I wanted to start making my portfolio. The development of Fingers Testing allowed me to gain practical experience in various aspects of game development, including programming, design, and testing. I learned basics of working in Unity, got familiar with its interface, functions and workflow. I got to practice my C# skills and learn how programing works in Unity. In addition, I honed my testing skills by designing and conducting tests to ensure the game's functionality.

What I learned during development:
  • RayCasting, Touch Inputs – for detecting where and what user is touching on the screen.
  • Instantiating – used for spawning targets in some of the minigames.
  • Creating and managing UI – like buttons or text. .
  • Working with TextMeshPro. .
  • Implementing basic save system with PlayerPrefs. .
  • Implementing Unity ads.
  • Building finished project.
  • Releasing project on Google Play Store.
Gravi Slime
  • Platform: Mobile
  • Available on: Android (Google Play Store)
  • Genre: Endless Runner
  • In Game Services: Ads (Rewarded), IAP
  • Link: play.google.com/store/apps/details?id=com.GrisDev.com.unity.GraviSlime.mobile2D
Gravi Slime was my second released game. It’s an endless runner where by touching the screen, player can change gravity and save slime from hitting spikes and walls. Game contains 20 pre-made level parts that will spawn randomly, so that every try will be different. I became adept at designing and implementing gameplay mechanics, such as the gravity manipulation system for the slime character, level generation, and score tracking. This allowed me to enhance my problem-solving abilities and develop creative solutions to challenges encountered during development. I created levels using Unity grid system and assets from Unity Asset Store. I also practiced performance optimalization using Unity profiler.

What I learned during development:
  • Unity 2D physics engine – colliders, rigidbody etc.
  • Using Unity grid system
  • Working with sound manager
  • Creating animations and their behavior
  • Implementing IAP buttons and connecting products to Google Play Console.
  • Implementing rewarded ads
  • Creating skin selection system
  • Creating prefabs
Ball POP
  • Platform: Mobile
  • Available on: Android (Google Play Store)
  • Genre: Color-matching Puzzle
  • In Game Services: IAP
  • Link: play.google.com/store/apps/details?id=com.DefaultCompany.com.unity.ballpop.mobile2D
Ball POP was third project that I finished and published. The goal of the game is gaining set amount of points by clicking connected balls with the same color – the more balls of the same color are connected and “POP-ed” the more points player gets. Balls are not set in place so player has to strategize to win. Whole Game is a variation of “Match 3” genre, while using gravity and collisions. Ball POP contains 5 level designs, random ball placements and score system that changes based on stage so that you are able to play it infinitely. One of the skills I learned from Ball POP was the importance of balancing game difficulty and maintaining player engagement that I tested by sharing the game with my friends and gaining their feedback. Through testing and balancing, I optimized the game's levels to ensure a smooth progression curve, gradually increasing the challenge while keeping players rewarded. This project was meant to mainly teach me creating engaging gameplay loops, implementing user feedback and designing intuitive interface. It also helped me practice creative solutions for programming gameplay mechanics.

What I learned during development:
  • Object pulling – for balls spawning
  • Creating simple 2D game assets – Inkscape
  • Using good programing practices
  • Keeping my code clean and easily readable
  • Implementing user feedback
Neon Space
  • Platform: Mobile
  • Available on: Android (Google Play Store)
  • Genre: Space Shooter, Survivor
  • In Game Services: Ads (Rewarded), IAP
  • Link: play.google.com/store/apps/details?id=com.grisdev.com.unity.neonspace.mobile2D
Neon Space is my latest mobile game that I developed. It’s based on Vampire Survivors game mechanics such as: starting from zero on death, upgrading and gaining new weapons by gaining levels, power up system based on in game currency and having big numbers of enemies coming at player at all times. Process of creating 8 individual weapons helped me practice creating mechanics using different approaches (list below). Optimalization was a big part of development because of large number of objects on screen, for that I used Unity profiler and some of code optimalization skills that I was learning during this time. Because of that I polished my skills concerning object pooling and implemented coroutines. I created all the graphics (apart from some UI elements that were Unity Store assets) in Inkscape and used Unity for animations. Neon Space was my biggest project at the time but thanks to it I learned many valuable skills and enhance already obtained.

What I learned during development:
  • Coroutines
  • Creating animations using Unity
  • Vectors – distance, move towards etc.
  • Line renderer – for “taser” weapon
  • Using time class
  • Creating leveling/upgrade system
  • Creating in game currency
  • Using Unity post processing asset
